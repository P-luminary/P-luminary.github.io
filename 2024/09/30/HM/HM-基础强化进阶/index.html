泛型类

类型变量建议用大写的英文字母：’E、T、K、V‘

在Java中，泛型类中的E、T、K、V 是常用的类型参数名称，它们各自有约定俗成的含义：

  • E (Element): 通常用于表示集合中的元素类型，如 List< E >。
  • T (Type): 通用类型参数，可以表示任何类型，如 Box< T >。
  • K (Key): 通常用于表示键值对中的键类型，如 Map< K, V >。
  • V (Value): 通常用于表示键值对中的值类型，如 Map< K, V >。

这些类型参数使得类或方法可以在编译时检查类型安全，并且所有的强制类型转换都是自动和隐式的，提高了代码的重用率和安全性。

详细体会多态概念

多态的概念：
多态是指同一个接口或基类的引用可以指向不同的实现类或子类的对象。
通过多态，你可以在不改变代码的情况下，使用不同的实现类来完成相同的操作。

// 创建集合 多态： 接口 变量名 = new 实现类();
Collection < String > coll = new ArrayList<>();

通过这种方式，coll 变量可以调用 Collection 接口中定义的所有方法，而具体的实现细节由 ArrayList 提供

迭代器遍历集合

  • 迭代器用来遍历集合的专用方式(数组没有迭代器)，在java中迭代器的代表是Iterator
方法名称 说明
Iterator< E > iterator() 返回集合中的迭代器对象，该迭代器对象默认指向当前集合的第一个元素
Iterator迭代器中的常用方法
方法名称 说明
boolean hasNext() 询问当前位置是否有元素存在，存在返回true 反之false
E next() 获取当前位置的元素，并同时将迭代器对象指向下一个元素处 
package 迭代器;

import java.util.ArrayList;
import java.util.Collection;
import java.util.Iterator;

public class demo12 {
    public static void main(String[] args) {
        // 创建Collection集合
        Collection<String> coll = new ArrayList<>();

        // 添加数据到集合中
        coll.add("赵敏");
        coll.add("小昭");
        coll.add("周芷若");
        coll.add("小華");

        // 使用迭代器获取集合中的数据
        // 1.获取集合的迭代器
        Iterator<String> it = coll.iterator();
        // 直接通过迭代器获取数据
        System.out.println("it.next() = " + it.next()); //赵敏
        System.out.println("it.next() = " + it.next()); //小昭
        System.out.println("it.next() = " + it.next()); //周芷若
        System.out.println("it.next() = " + it.next()); //小華

        System.out.println("=========================");

        Iterator<String> its = coll.iterator();
        // 循环判断是否有下一个元素 有就取出
        while (its.hasNext()) { // true or false
            // 获取下一个元素
            String name = its.next();
            System.out.println("name = " + name);
        }
    }
}

为什么要用迭代器 什么场景用 为什么不用for

  • 为什么要用迭代器：
    灵活性：迭代器提供了一种统一的方式来遍历不同类型的集合，而不需要关心集合的具体实现。
    功能丰富：迭代器不仅提供了遍历集合的功能，还提供了删除元素等操作。

  • 什么场景用迭代器：
    多态性：当需要遍历多种类型的集合时，使用迭代器可以避免重复代码。
    并发修改：在遍历集合的同时需要修改集合（如删除元素），使用迭代器更安全。
    自定义集合：对于自定义的集合类，可以通过实现迭代器接口来提供遍历功能。

  • 为什么不用for：
    类型限制：增强的 for 循环（即 for-each 循环）只能用于遍历集合或数组，不能进行其他操作（如删除元素）。
    灵活性不足：增强的 for 循环不适用于所有集合类型，特别是那些不支持随机访问的集合（如 LinkedList）。
    并发修改问题：在使用增强的 for 循环遍历集合时，如果集合被修改，会抛出 ConcurrentModificationException。

增强For循环

for(元素的数据类型 变量名 ：数组或集合){

}

  • 增强for可以用来遍历集合或者数组
  • 代码简单方便、没有索引
  • 增强for循环遍历集合底层原理：迭代器 [迭代器可查看]
package 增强for循环;

import java.util.Collection;

public class demo01 {

        coll.add("小華");

        // 使用增强for遍历集合中的数据

        System.out.println("=========================");
        String[] arr = {"小明", "小红", "小花"};
        for (String name : arr) {
            System.out.println("name = " + name);
        }
    }
}

List集合特点(ArrayList)
  • 有序、可重复、有索引
ArrayList集合原理
  • 基于数组实现的
  • 查询速度快：(根据索引查询数据快)
  • 删除效率低：要把后面很多的数据进行前移
  • 添加效率低：后面很多数据后移，再添加元素；也可能需要数组扩容
LinkedList底层原理
  • LinkedList底层是基于链表存储数据的
  • 底层为双链表实现
链表的特点
  • 链表中的数据是一个个独立的结点组成的，结点在内存中是不连续的，每个结点包含数据值和下一个结点的地址。
  • 查询效率慢，无论查询哪个数据都要从头开始找
  • 链表增删相对快

        System.out.println("添加首位后 list = " + list);

        list.removeFirst();
        list.removeLast();
        // [刘德华, 张学友, 郭富城, 黎明]
        System.out.println("删除首位后 list = " + list);

HashSet集合的底层原理
  • 基于哈希表存储数据的
  • 哈希表是一种增删改查数据 都很好用！
  • HashSet在添加元素时内部会根据**(哈希算法)计算元素应存入的位置**
  • 模拟哈希算法：(存入位置 = 哈希值 % 数组长度) 余数肯定在 [0-数组长度) 之间
    [hashcode(决定位置) + equals(决定能否存)]
    • 先根据hashCode值计算元素存放的位置
    • 如果这个位置没有元素，直接存储
    • 如果这个位置有元素，调用equals比较 “重地”与”通话”
      (hashcode一样但是equals不一样 可以存储 新元素直接挂在老元素下面)
      • JDK 8 之前，新元素存入数组，占老元素位置，老元素挂在下面【头插法】
      • JDK 8开始之后，新元素直接挂在老元素下面
      • JDK 8开始之后，当链表长度超过8，数组长度>=64，自动将链表转成红黑树

package HashSet集合实例;

import lombok.AllArgsConstructor;
import lombok.Data;
import lombok.NoArgsConstructor;

import java.util.Objects;

//@Data
//@NoArgsConstructor
//@AllArgsConstructor
public class Student {
    private String name;
    private int age;
    private double score;

    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (o == null || getClass() != o.getClass()) return false;

        Student student = (Student) o;

        if (age != student.age) return false;
        if (Double.compare(student.score, score) != 0) return false;
        return Objects.equals(name, student.name);
    }

    @Override
    public int hashCode() {
        int result;
        long temp;
        result = name != null ? name.hashCode() : 0;
        // 31是质数 乘法相同的概率最低
        result = 31 * result + age;
        temp = Double.doubleToLongBits(score);
        result = 31 * result + (int) (temp ^ (temp >>> 32));
        return result;
    }

    @Override
    public String toString() {
        return "Student{" +
                "name='" + name + '\'' +
                ", age=" + age +
                ", score=" + score +
                '}';
    }
    public Student() {
    }

    public Student(String name, int age, double score) {
        this.name = name;
        this.age = age;
        this.score = score;
    }
// Getter+Setter
}

LinkedHashSet底层原理

  • 依旧是基于哈希表(数组、链表、红黑树)实现的
  • 它的每个元素都额外多了一个双链表的机制记录它前后元素的位置
    • 有序、不重复、无索引
    • 底层基于哈希表，使用双链表记录数据

TreeSet集合原理

  • 特点：不重复、无索引、一定排序(默认升序排序 从小到大)
  • 底层是基于红黑树实现的排序
  • 字符串按照首字母[ASCII]值 数字就从小到大排序

自定义排序规则

HashSet (实现)→(重写) Hashcode ＋ Equals
TreeSet (实现)→(重写) CompareTo

什么时候用什么？

如果希望记住元素的添加顺序，需要存储重复的元素，又要频繁的根据索引查询数据？
  • 用ArrayList集合（有序、可重复、有索引），底层基于数组的。（常用）
如果希望记住元素的添加顺序，且增删首尾数据的情况较多？
  • 用LinkedList集合（有序、可重复、有索引），底层基于双链表实现的。
如果不在意元素顺序，也没有重复元素需要存储，只希望增删改查都快？
  • 用HashSet集合（无序，不重复，无索引），底层基于哈希表实现的。 （常用）
如果希望记住元素的添加顺序，也没有重复元素需要存储，且希望增删改查都快？
  • 用LinkedHashSet集合（有序，不重复，无索引）， 底层基于哈希表和双链表
如果要对元素进行排序，也没有重复元素需要存储？且希望增删改查都快？
  • 用TreeSet集合，基于红黑树实现。

TreeSet集合存储自定义类型对象

  • 让自定义的类（如学生类）实现Comparable接口，重写里面的compareTo方法来指定比较规则。
  • 通过调用TreeSet集合有参数构造器，可以设置Comparator对象（比较器对象，用于指定比较规则

}
