异常/自定义异常/ExceptionByMyself.java
package 异常.自定义异常;

import java.util.Scanner;

public class ExceptionByMyself {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        System.out.println("请输入姓名：");
        String name = sc.next();
        System.out.println("请输入年龄：");
        int age = sc.nextInt();
        try {   // 受到异常
            register(name, age);
        } catch (U18Exception e) {
            System.out.println("年龄不合法，请使用父母的身份证");
        } finally {
            System.out.println("finally代码块中的代码，总会被执行");
        }
    }
    // 模拟网站注册
    public static void register(String name, int age) throws U18Exception {
        if (age < 18) { // 产生异常
    // 小于18不让注册, 产生异常, 抛出异常, 让调用的地方去处理
            throw new U18Exception("年龄不能小于18");
        } else {
            System.out.println("注册成功");
        }
    }
}

return：结束/回调方法
finally：稳妥一点，一定执行，把资源释放 (关闭文件、IO流关闭) 的代码放到finally中

泛型类

类型变量建议用大写的英文字母：’E、T、K、V‘

在Java中，泛型类中的E、T、K、V 是常用的类型参数名称，它们各自有约定俗成的含义：

  • E (Element): 通常用于表示集合中的元素类型，如 List< E >。
  • T (Type): 通用类型参数，可以表示任何类型，如 Box< T >。
  • K (Key): 通常用于表示键值对中的键类型，如 Map< K, V >。
  • V (Value): 通常用于表示键值对中的值类型，如 Map< K, V >。

这些类型参数使得类或方法可以在编译时检查类型安全，并且所有的强制类型转换都是自动和隐式的，提高了代码的重用率和安全性。

泛型类/MyArrayList.java
package 泛型类;

// 1.<E>：定义泛型，相当于定义变量
// 2. 把具体的类型修改为E
public class MyArrayList<E> {
    // 添加元素
    public void add(E e) {

    }

    // 获取元素
    public E get(){
        return null;
    }
}

  • 集合是一种容器，类似于数组，但集合的大小可改变
    • Collection 单列集合 每个元素只包含一个值
    • Map 双列集合 每个元素包含两个值(键值对)

  • List系列集合：添加的元素是有序、可重复、有索引 [增删改查]
  • Set系列集合：添加的元素是无序、不重复、无索引
Collection的常用功能
  • Collection是单列集合的祖宗，它规定的方法(功能)是全部单列集合都会继承的

详细体会多态概念

多态的概念：
多态是指同一个接口或基类的引用可以指向不同的实现类或子类的对象。
通过多态，你可以在不改变代码的情况下，使用不同的实现类来完成相同的操作。

// 创建集合 多态： 接口 变量名 = new 实现类();
Collection < String > coll = new ArrayList<>();

通过这种方式，coll 变量可以调用 Collection 接口中定义的所有方法，而具体的实现细节由 ArrayList 提供

迭代器遍历集合

  • 迭代器用来遍历集合的专用方式(数组没有迭代器)，在java中迭代器的代表是Iterator
方法名称 说明
Iterator< E > iterator() 返回集合中的迭代器对象，该迭代器对象默认指向当前集合的第一个元素
Iterator迭代器中的常用方法
方法名称 说明
boolean hasNext() 询问当前位置是否有元素存在，存在返回true 反之false
E next() 获取当前位置的元素，并同时将迭代器对象指向下一个元素处 
package 迭代器;

import java.util.ArrayList;
import java.util.Collection;
import java.util.Iterator;

public class demo12 {
    public static void main(String[] args) {
        // 创建Collection集合
        Collection<String> coll = new ArrayList<>();

        // 添加数据到集合中
        coll.add("赵敏");
        coll.add("小昭");
        coll.add("周芷若");
        coll.add("小華");

        // 使用迭代器获取集合中的数据
        // 1.获取集合的迭代器
        Iterator<String> it = coll.iterator();
        // 直接通过迭代器获取数据
        System.out.println("it.next() = " + it.next()); //赵敏
        System.out.println("it.next() = " + it.next()); //小昭
        System.out.println("it.next() = " + it.next()); //周芷若
        System.out.println("it.next() = " + it.next()); //小華

        System.out.println("=========================");

        Iterator<String> its = coll.iterator();
        // 循环判断是否有下一个元素 有就取出
        while (its.hasNext()) { // true or false
            // 获取下一个元素
            String name = its.next();
            System.out.println("name = " + name);
        }
    }
}

为什么要用迭代器 什么场景用 为什么不用for

  • 为什么要用迭代器：
    灵活性：迭代器提供了一种统一的方式来遍历不同类型的集合，而不需要关心集合的具体实现。
    功能丰富：迭代器不仅提供了遍历集合的功能，还提供了删除元素等操作。

  • 什么场景用迭代器：
    多态性：当需要遍历多种类型的集合时，使用迭代器可以避免重复代码。
    并发修改：在遍历集合的同时需要修改集合（如删除元素），使用迭代器更安全。
    自定义集合：对于自定义的集合类，可以通过实现迭代器接口来提供遍历功能。

  • 为什么不用for：
    类型限制：增强的 for 循环（即 for-each 循环）只能用于遍历集合或数组，不能进行其他操作（如删除元素）。
    灵活性不足：增强的 for 循环不适用于所有集合类型，特别是那些不支持随机访问的集合（如 LinkedList）。
    并发修改问题：在使用增强的 for 循环遍历集合时，如果集合被修改，会抛出 ConcurrentModificationException。
