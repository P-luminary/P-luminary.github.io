阿里巴巴领域模型DTO和VO介绍

接收对象和返回对象分别使用了DTO和VO(阿里巴巴的领域模型)

DTO：Data Transfer Obiect数据传输对象:xxxDto或者xxxDTO，xxx为业务领域相关的名称。接口的入参并传递给业务层
VO：Value Object(View Object)视图对象:xxxVO或者xxxVo，xxx一般为网页名称。接口的出参，一般封装0在通用结果data属性中返回给前端
Entity：普通实体类，属性与数据库表–对应，一般操作持久层使用

后端开发流程和接口设计规范

5.2.1 需求分析

在接收到任务之后，我们需要查看任务涉及到的模块，需要快速熟悉业务，熟悉业务的参考就是原型文档和PRD文档，我们接下来要开发的床位相关的接口，那么就详细查看床位相关的原型图即可。
特殊情况，新需求可能会没有原型图，这时就需要跟产品经理要多沟通，直至搞明白需求为止

5.2.2 接口四要素

搞明白需求之后，我们下面就可以来设计接口了，一个接口包含了四个基本要素，分别是：请求路径、请求方式、接口入参(请求参数)、接口出参(响应数据)

  • 请求路径 命名：以模块名称进行区分（英文）
  • 请求方式（需要符合restFul风格）
    • 查询 GET
    • 新增 POST
    • 修改 PUT
    • 删除 DELETE
  • 接口入参(请求参数)
    • 路径上的参数
      • query传参—->后端形参接收 url?key1=value1&key2=value2…
        查询 通过id查询 通过名称查询 分页查询
      • path传参—->后端PathVariable注解接收 url/value1/value2/…
        删除/查询
    • 请求体参数
      新增和修改用JSON参数
      • 前端：json对象字符串
      • 后端：对象接收，DTO
  • 接口出参(响应参数)
    • 统一格式 {code:200,msg:"成功",data:{}}
    • 数据封装，一般为VO
      • 敏感数据过滤
      • 整合数据

package com.zzyl.controller;

import com.zzyl.base.ResponseResult;
import com.zzyl.dto.BedDto;
import com.zzyl.service.BedService;
import com.zzyl.vo.BedVo;
import io.swagger.annotations.Api;
import io.swagger.annotations.ApiOperation;
import io.swagger.annotations.ApiParam;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.web.bind.annotation.*;

import java.util.List;

@RestController
@RequestMapping("/bed")
@Api(tags = "床位管理相关接口")
public class BedController extends BaseController {

    @Autowired
    private BedService bedService;

    @GetMapping("/read/room/{roomId}")
    @ApiOperation(value = "根据房间id查询床位", notes = "传入房间id")
    public ResponseResult<List<BedVo>> readBedByRoomId(
            @ApiParam(value = "房间ID", required = true) @PathVariable("roomId") Long roomId) {
        List<BedVo> beds = bedService.getBedsByRoomId(roomId);
        return success(beds);
    }

    /**
     * 创建床位
     * @param bedDto
     * @return
     */
    @PostMapping("/create")
    public ResponseResult addBadd(@RequestBody BedDto bedDto) {
        // 调用业务层创建床位  logger直接在controller里继承了BaseController
        logger.info("bedDto: {}", bedDto);
        bedService.addBed(bedDto);
        return success();
    }
}

通过注解可以控制生成的接口文档，使接口文档拥有更好的可读性，常用注解如下：

注解 说明
@Api 用在控制器类上，描述Controller的作用
@ApiOperation 用在控制器方法上，说明方法的用途、作用
@ApiParam 用在方法的参数上，描述单个形参的含义
@ApiImplicitParam 用在方法上，描述单个形参的含义
@ApiModel 用在类上，用对象来接收参数或者返回参数，描述类的含义
@ApiModelProperty 用在类的属性上，用对象来接收参数或者返回参数，描述字段的含义
@ApiOperationSupport 用在方法上，用于描述请求参数对象里面哪些属性需要传递@ApiOperationSupport(includeParameters = {“参数对象名.属性名1”,”参数对象名.属性名2”,”参数对象名.属性名3”,…})

如何评估工期

评估工期是程序员的必备的一个技能，评估的准确与否，一定程度上会影响在当前团队发展的边界。

那如何做工期评估呢？

当我们被分配到需要开发某个模块之后，我们从以下几个方面来评估工期：

暂时无法在飞书文档外展示此内容

  • 需求分析
    • 首先需要对需求进行充分的分析和理解，包括功能、性能、安全等方面的需求。确保对需求的理解准确，避免后期需求变更导致工期延误。
  • 技术复杂度
    • 评估所涉及的技术栈和相关技术难点，包括数据库设计、性能优化、安全防护等方面的复杂度。技术复杂度高的项目通常需要更多的时间来完成，如果个人之前没有涉及过，还需要花时间去学习。
  • 任务分解
    • 将整体任务分解为具体的模块或功能点，对每个模块进行评估，估算每个模块的开发工期。
  • 风险评估：
    • 评估项目中可能出现的风险和问题，如第三方依赖、需求变更、人力资源等方面的风险，合理考虑这些风险对工期的影响。
  • 团队协作
    • 评估模块完成需要团队其他人来共同协助开发，比如模块间有依赖关系，接口需要联调、功能需要测试等等
  • 沟通
    • 提前沟通、经常沟通，了解彼此的日程和需求变更，团队内信息要及时同步与更新

  • 护理项目作为护理模块的基础数据，主要是由护理员给老年人提供护理服务，常见的有：洗头、助餐、复健运动等，护理项目分为护理计划内与护理计划外，两者之间的区别是：前者是根据护理计划产生的，后者是由家属从用户端进行下单支付产生的

E-R图画ER图的时候要严格根据项目原型和右面的开发规则去对应[实体]和[属性]

实体-关系图，提供了表示实体(类，事务)类型、属性(成员方法，事务的数据) 和 关联的方法

假设我们要设计一个简单的图书馆数据库系统。我们可以使用E-R图来描述这个系统的数据模型

  1. 确定有哪些实体？书 作者 会员
  2. 每个实体之间的关系是什么？书和作者多对多 书和用户多对多
  3. 每个实体自己的属性有哪些？

有以下实体（Entities）：

  • 书籍（Book）
  • 作者（Author）
  • 图书馆成员（Library Member）(理解为借书用户)

实体之间的关系（Relationships）：

  • 一本书可以由一个或多个作者编写，一个作者可以编写多本书，这是一个“多对多”的关系。
  • 图书馆成员可以借阅多本书，而每本书也可以被多个图书馆成员借阅，这是一个“多对多”的关系。

每个实体包含的属性（Attributes）：

  • 书籍实体可能包括书名、ISBN号、出版日期等属性。
  • 作者实体可能包括姓名、国籍、出生日期等属性。
  • 图书馆成员实体可能包括会员号、姓名、联系信息等属性。

通过E-R图，我们可以清晰地展示这些实体之间的关系和它们的属性，从而更好地理解图书馆数据库系统的数据模型。

详细设计(如何设计表中有哪些字段)

  • 命名（见名之意） 参考阿里开发手册
  • 数据类型
    • 基本准则
      • 可靠性：考虑字段存储的内容长度，尽可能的合适
      • 存储空间：在考虑可靠性的前提下，再次考虑存储空间
    • 常见数据类型 Day02-MySQL常见数据类型
  • 主键（必须存在）
    • 自增
    • UUID
    • 雪花算法
  • 参照产品页面原型设计字段（核心业务字段）
  • 冗余字段
    • 重复的字段，其他表已经有这个字段，当前表又有
    • 例子
      • 订单表：商品价格（冗余字段，因为商品表里面有了，这个价格记录的是当时购买的价格，而且以后不会发生变化，不随商品价格变化）
      • 商品表：商品价格，这里的价格根据商家自己的不同时间段，会对价格进行不同调整
  • 公共字段
    • 创建时间
    • 修改时间
    • 创建人
    • 修改人
    • 备注

    @Override
    public void add(NursingPlanDto nursingPlanDto) {
        NursingPlan nursingPlan = BeanUtil.copyProperties(nursingPlanDto, NursingPlan.class);
        nursingPlan.setUpdateTime(LocalDateTime.now());
        nursingPlan.setCreateTime(LocalDateTime.now());
        nursingPlan.setCreateBy(10L);
        nursingPlan.setUpdateBy(10L);
        nursingPlan.setStatus(0);
        nursingPlanMapper.add(nursingPlan);
        Long id = nursingPlan.getId();
        List<NursingProjectPlanDto> projectPlans = nursingPlanDto.getProjectPlans();
        projectPlans.forEach(nursingProjectPlanDto -> nursingProjectPlanDto.setPlanId(id));
        nursingProjectPlanMapper.addProjectPlan(projectPlans);
    }

    /**
     * 根据id查询护理计划
     *
     * @param id
     * @return
     */
    @Override
    @Transactional
    public NursingPlanVo getById(Long id) {
        //根据id查询护理项目
        NursingPlan nursingPlan = nursingPlanMapper.getById(id);
        NursingPlanVo nursingPlanVo = new NursingPlanVo();
        BeanUtils.copyProperties(nursingPlan, nursingPlanVo);
        //根据id查询护理计划项目
        List<NursingProjectPlanVo> nursingProjectPlanVoList = nursingProjectPlanMapper.getById(id);
        if (nursingProjectPlanVoList != null) {
            //给nursingPlanVo的projectPlans设值
            nursingPlanVo.setProjectPlans(nursingProjectPlanVoList);
        }
        return nursingPlanVo;
    }

    /**
     * 修改护理计划
     *
     * @param nursingPlanDto
     */
    @Override
    @Transactional
    public void updateById(Long id, NursingPlanDto nursingPlanDto) {
        // 查询是否关联护理等级表
        nursingPlanMapper.updateById(nursingPlanDto);
        // 删除护理计划项目
        nursingProjectPlanMapper.deleteById(id);
        // 根据id查询护理计划项目
        List<NursingProjectPlanVo> nursingProjectPlanVoList = nursingProjectPlanMapper.getById(id);
        // 判断projectPlans是否为空
        if (nursingProjectPlanVoList != null) {
            List<NursingProjectPlanDto> projectPlans = nursingPlanDto.getProjectPlans();
            projectPlans.forEach(projectPlan -> {
                projectPlan.setPlanId(id);
            });
            //循环插入计划项目
            nursingProjectPlanMapper.addProjectPlan(projectPlans);
        }
    }

    /**
     * 删除护理计划项目
     *
     * @param id
     */
    @Override
    @Transactional
    public void deleteById(Long id) {
        //状态为1的不能删除
        NursingPlan nursingPlan = nursingPlanMapper.getById(id);
        if (nursingPlan.getStatus() == 1) {
            throw new RuntimeException("状态启用中,不能删除!");
        }
        //关联护理等级的不能删除

        //根据id删除护理计划
        nursingPlanMapper.deleteById(id);

        //根据id查询护理计划项目
        List<NursingProjectPlanVo> nursingProjectPlanVoList = nursingProjectPlanMapper.getById(id);
        //判断是否为空
        if (nursingProjectPlanVoList != null) {
            //删除护理计划项目
            nursingProjectPlanMapper.deleteById(id);
        }
    }

    /**
     * 修改护理计划状态
     *
     * @param id
     * @param status
     */
    @Override
    public void updateStatus(Long id, Integer status) {
        nursingPlanMapper.updateStatus(id, status);
    }

     * 查询所有护理计划

     * @param nursingPlan
     * @return
     */
    @Override
    public List<NursingPlan> list(NursingPlan nursingPlan) {
        List<NursingPlan> nursingPlanList = nursingPlanMapper.list(nursingPlan);
        return nursingPlanList;
    }

package com.zzyl.intercept;

import cn.hutool.core.util.ObjectUtil;
import com.zzyl.utils.EmptyUtil;
import com.zzyl.utils.UserThreadLocal;
import org.apache.ibatis.binding.MapperMethod;
import org.apache.ibatis.executor.Executor;
import org.apache.ibatis.mapping.MappedStatement;
import org.apache.ibatis.mapping.SqlCommandType;
import org.apache.ibatis.plugin.*;
import org.apache.ibatis.reflection.MetaObject;
import org.apache.ibatis.reflection.SystemMetaObject;
import org.springframework.stereotype.Component;

import java.time.LocalDateTime;
import java.util.ArrayList;
import java.util.Properties;

@Intercepts({
        @Signature(type = Executor.class, method = "update", args = {MappedStatement.class, Object.class})
})
public class AutoFillInterceptor implements Interceptor {

    private static final String CREATE_BY = "createBy";
    private static final String UPDATE_BY = "updateBy";

    private static final String CREATE_TIME = "createTime";
    private static final String UPDATE_TIME = "updateTime";

    @Override
    public Object intercept(Invocation invocation) throws Throwable {
        Object[] args = invocation.getArgs();
        // 获取用于描述SQL语句的映射信息
        MappedStatement ms = (MappedStatement) args[0];
        SqlCommandType sqlCommandType = ms.getSqlCommandType();

        // 获取sql参数实体 ParamMap
        Object parameter = args[1];

        if (parameter != null && sqlCommandType != null) {
            // 获取用户ID
            Long userId = loadUserId();

            if (SqlCommandType.INSERT.equals(sqlCommandType)) {
                // 插入操作
                if (parameter instanceof MapperMethod.ParamMap) {
                    // 批量插入的情况
                    MapperMethod.ParamMap paramMap = (MapperMethod.ParamMap) parameter;
                    ArrayList list = (ArrayList) paramMap.get("list");
                    list.forEach(v -> {
                        // 设置创建人和创建时间字段值
                        setFieldValByName(CREATE_BY, userId, v);
                        setFieldValByName(CREATE_TIME, LocalDateTime.now(), v);
                        setFieldValByName(UPDATE_TIME, LocalDateTime.now(), v);
                    });
                    paramMap.put("list", list);
                } else {
                    // 单条插入的情况
                    // 设置创建人和创建时间字段值
                    setFieldValByName(CREATE_BY, userId, parameter);
                    setFieldValByName(CREATE_TIME, LocalDateTime.now(), parameter);
                    setFieldValByName(UPDATE_TIME, LocalDateTime.now(), parameter);
                }
            } else if (SqlCommandType.UPDATE.equals(sqlCommandType)) {
                // 更新操作
                // 设置更新人和更新时间字段值
                setFieldValByName(UPDATE_BY, userId, parameter);
                setFieldValByName(UPDATE_TIME, LocalDateTime.now(), parameter);
            }
        }

        // 继续执行原始方法
        return invocation.proceed();
    }

    /**
     * 通过反射设置实体的字段值
     * @param fieldName 字段名
     * @param fieldVal  字段值
     * @param parameter 实体对象
     */
    private void setFieldValByName(String fieldName, Object fieldVal, Object parameter) {
        MetaObject metaObject = SystemMetaObject.forObject(parameter);
       
        if (metaObject.hasSetter(fieldName)) {
            metaObject.setValue(fieldName, fieldVal);
        }
    }

    @Override
    public Object plugin(Object target) {
        if (target instanceof Executor) {
            // 对目标对象进行包装，返回代理对象
            return Plugin.wrap(target, this);
        }
        // 非 Executor 类型的对象，直接返回原始对象
        return target;
    }

    @Override
    public void setProperties(Properties properties) {
        // 读取配置文件中的属性，此处没有使用
    }

    /**
     * 获取当前用户的ID，用于填充创建人和更新人字段的值
     *
     * @return 当前用户ID
     */
    public static Long loadUserId() {
        // 从 ThreadLocal 中获取用户ID
        Long userId = UserThreadLocal.getUserId();
        // 如果 ThreadLocal 中不存在用户ID，则从管理用户ID中获取
        if (ObjectUtil.isNotEmpty(userId)) {
            return userId;
        }
        userId = UserThreadLocal.getMgtUserId();
        // 如果管理用户ID也不存在，则默认返回ID为1的用户
        if (!EmptyUtil.isNullOrEmpty(userId)) {
            return userId;
        }
        return 1L;
    }
}

小程序登录校验

接口文档

我们在项目中对接任何的第三方接口，最重要的参考永远都是官方给提供的接口文档。所以大家一定要具备阅读接口文档的能力才能更好的对接接口使用。

那接口文档该怎么阅读呢？

其实跟我们定义的接口差不了太多，也必然会包含接口的四要素（请求路径、请求方式、入参、出参）

我们来看一下微信官方给提供的这个接口，这个是用于微信登录的接口，链接如下：

https://developers.weixin.qq.com/miniprogram/dev/OpenApiDoc/user-login/code2Session.html

这个文档中比较关键的部分就是：

http和https的区别

  • http是明文传输 请求和相应 【内网可以走】
  • https是密文传输 非对称的加解密来保证其安全

请求参数

属性 类型 必填 说明
appid string 是 小程序 appId
secret string 是 小程序 appSecret
js_code string 是 登录时获取的 code，可通过wx.login获取
grant_type string 是 授权类型，此处只需填写 authorization_code

返回参数

属性 类型 说明
session_key string 会话密钥
unionid string 用户在开放平台的唯一标识符，若当前小程序已绑定到微信开放平台账号下会返回，详见 UnionID 机制说明。
errmsg string 错误信息
openid string 用户唯一标识

修复 Bug：如果你发现护理项目创建功能有问题，可以运行这个测试方法来复现问题并进行调试。

以上两个方法调用相对简单，如果请求参数中没有额外的条件设置，就可以使用上述的两个方法，但是，有的时候三方接口调用需要使用到header或者是文件上传，则不能使用上述两个方法

我们可以使用HttpUtil.*createRequest*方法来去构建更多的条件参数

使用 HttpUtil.createRequest(Method.POST, url).body(JSONUtil.toJsonStr(paramMap)).execute(); 而不是 HttpUtil.post(url, JSONUtil.toJsonStr(paramMap)); 有以下几点优点和好处：
灵活性更高：
createRequest 方法提供了更多的配置选项，例如设置请求头、超时时间、重试机制等。这对于复杂的请求场景非常有用。
你可以更细粒度地控制请求的各个方面，而不仅仅是发送一个简单的 POST 请求。
可读性和可维护性：
使用链式调用的方式使得代码更加清晰易读，每一步操作都一目了然。
如果未来需要添加更多的配置或处理逻辑，使用 createRequest 方式更容易扩展和维护。
错误处理更灵活：
createRequest 返回的是一个 HttpRequest 对象，你可以在这个对象上添加更多的错误处理逻辑，例如捕获异常、重试请求等。
例如，你可以使用 .timeout(5000) 设置超时时间，或者使用 .header("Content-Type", "application/json") 设置请求头。
调试更方便：
在请求执行之前，你可以插入断点或日志输出，检查请求的各个部分是否符合预期。
例如，你可以在 .body(JSONUtil.toJsonStr(paramMap)) 之后插入日志输出，查看请求体的内容。
